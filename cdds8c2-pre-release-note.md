### SAT (Boolean Satisfiability Problem)

> одна из фундаментальных NP-полных проблем в информатике.

Для практических приложений (верификация схем, планирование, криптоанализ) критически важна эффективность решателей в **трудных режимах** около фазового перехода, где классические эвристики деградируют.

`Random 3-SAT` инстансы с отношением `m/n ≈ 4.2-4.5` (фазовый переход) представляют особую сложность для `stochastic local search (SLS)` алгоритмов. При ограниченном бюджете шагов (например, 180 флипов) современные решатели застревают в локальных оптимумах, не достигая 100% удовлетворенных клауз.

`prototype algorithm 8C2` развивает идеи `CSAW (Clause-Selection And Weighting)` и адаптивного scoring, добавляя три ключевых компонента: формальную primal-dual динамику весов клауз, size/density-aware режимы для разных размеров задач и улучшенную обработку zero-break флипов. Результат: **на 0.04-0.19% больше удовлетворенных клауз** по сравнению с probSAT и WalkSAT при фиксированном бюджете шагов.

Для инстансов с n=3000 и m≈12800 клауз разница в 0.19% означает около **24 дополнительных удовлетворенных клаузы** — существенный выигрыш для hard-режимов.

## CNF-формула (Conjunctive Normal Form)

$$
F = \{C_1, C_2, \dots, C_m\}
$$

где каждая клауза $C_i$ — это дизъюнкция литералов над $n$ булевыми переменными.

задача найти присваивание $x \in \{0,1\}^n$, максимизирующее число удовлетворенных клауз:

$$
\max_x \; S(x) = \sum_{i=1}^{m} \mathbf{1}[C_i(x) = \text{true}]
$$

где $\mathbf{1}[\cdot]$ — индикаторная функция (1 если условие истинно, 0 иначе).

**Частный случай SAT:** если $S(x) = m$, все клаузы удовлетворены → инстанс satisfiable.

### Метрика качества для hard-режимов

При ограниченном бюджете шагов (например, 180 флипов) ключевая метрика:

$$
\text{Success Rate} = \frac{S(x)}{m} \times 100\%
$$

Для конкурентного сравнения алгоритмов мы фиксируем размер задачи через $n$ (число переменных), плотность через $\alpha = m/n$ (отношение клауз к переменным) и бюджет как фиксированное число флипов (обычно 180).

## Архитектура prototype algorithm 8C2

### Общая схема (она не отображает чувствительных деталей алгоритма в силу авторского права)

prototype algorithm 8C2 остается однофлипным `stochastic local search` алгоритмом, но использует несколько взаимодополняющих компонентов.

![Схема пайплайна prototype algorithm 8C2 в тёмной теме](/img/prototype-8c2/pipeline.svg)

| Компонент                     | Назначение                                              |
| ----------------------------- | ------------------------------------------------------- |
| **Adaptive scoring**          | Динамический расчет приоритетов флипов через make/break |
| **Clause weighting**          | Динамические веса конфликтных клауз (dense-gated режим) |
| **Size/density awareness**    | Адаптация параметров к размеру задачи                   |
| **Zero-break prioritization** | Отдельная обработка "бесплатных" ходов                  |

### Make/Break Scoring

Для каждого кандидата $v$ внутри выбранной UNSAT-клаузы вычисляется **вероятность флипа**:

$$
p(v) \propto
\bigl(1 + \alpha \cdot \text{make}(v)\bigr)
\cdot
\bigl(\varepsilon + \text{break}(v) + \text{novelty}(v)\bigr)^{-c_b}
$$

![Разложение make/break scoring с zero-break веткой](/img/prototype-8c2/make-break-scoring.svg)

Величина $\text{make}(v)$ определяет число клауз, которые **станут** удовлетворены после флипа $v$, в то время как $\text{break}(v)$ показывает число клауз, которые **перестанут** быть удовлетворены. Параметр $\text{novelty}(v)$ добавляет штраф за слишком недавний флип этой переменной. Коэффициент $\alpha$ (обозначается как `make_weight`) контролирует вес награды за make, а экспонента $c_b$ (параметр `cb`) определяет силу штрафа за break. Малая константа $\varepsilon$ обеспечивает численную стабильность формулы.

**Специальный случай:** Zero-break кандидаты (переменные с `break(v) = 0`) выбираются **случайно** среди себе подобных, чтобы не терять "бесплатные" улучшения.

Формула балансирует между максимизацией новых удовлетворенных клауз (make), минимизацией потерь (break) и диверсификацией поиска (novelty).

## Primal-Dual обновление весов клауз

### когда мы активируем dense-режим?

Dense-gated primal-dual включается при:

$$
\frac{m}{n} \geq \text{dual\_density\_on}
$$

В плотных задачах (много клауз на переменную) простое равномерное weighting неэффективно. Формальная динамика весов обеспечивает более точную адаптацию к структуре задачи.

### Формула обновления веса

Для выбранной клаузы $C_i$ на шаге $t$:

$$
\eta_t = \frac{\eta}{\sqrt{t+1}}
$$

$$
g_t = \mathbf{1}[C_i \text{ остается UNSAT после флипа}]
$$

$$
w_i^{t+1} = \Pi_{[w_{\min}, w_{\max}]}
\left(
w_i^t + \eta_t \bigl(g_t - \lambda (w_i^t - 1)\bigr)
\right)
$$

![Схема primal-dual обновления весов клауз](/img/prototype-8c2/primal-dual-update.svg)

Параметр $\eta_t$ представляет собой **адаптивный шаг обучения**, который уменьшается со временем для стабилизации процесса. Величина $g_t$ является **градиентом**, принимающим значение 1 если клауза осталась неудовлетворенной и 0 в противном случае. Коэффициент $\lambda$ отвечает за **регуляризацию** к весу 1.0 и предотвращает разрастание весов. Оператор $\Pi_{[w_{\min}, w_{\max}]}$ выполняет **clipping** результата в допустимый диапазон.

Если клауза упорно остается UNSAT, ее вес растет, и она чаще выбирается на следующих итерациях. Регуляризация не дает весам "убежать в бесконечность", поддерживая их в разумных пределах. Уменьшающийся шаг $\eta_t$ делает процесс более стабильным на поздних стадиях, когда алгоритм уже близок к хорошему решению.

**Связь с теорией оптимизации:** Это вариант subgradient descent для дуальной задачи MAX-SAT с адаптивной регуляризацией.

## Экспериментальные результаты

### Верхний диапазон размеров (n=1500-3000)

Эксперименты проводились с плотностью `ratio = 4.267` (около фазового перехода), бюджетом `steps = 180` на инстансах random 3-SAT.

**Success Rate (% удовлетворенных клауз)**

| n    | prototype algorithm 8C2 | probSAT | W-WalkSAT | Δ vs probSAT | Δ vs W-WalkSAT |
| ---- | ----------------------: | ------: | --------: | -----------: | -------------: |
| 1500 |             **99.8359** | 99.7422 |   99.6641 |     +0.09375 |       +0.17188 |
| 2000 |             **99.8506** | 99.7451 |   99.6836 |     +0.10546 |       +0.16698 |
| 2500 |             **99.8383** | 99.7984 |   99.6625 |     +0.03984 |       +0.17578 |
| 3000 |             **99.8164** | 99.7695 |   99.6270 |     +0.04687 |       +0.18944 |

prototype algorithm 8C2 **стабильно лидирует** на всех размерах, демонстрируя преимущество над probSAT в диапазоне **0.04-0.11%** и над W-WalkSAT в диапазоне **0.16-0.19%**. При n=3000 разница в 0.19% эквивалентна примерно **24 дополнительным удовлетворенным клауза**, что представляет существенный выигрыш для hard-режимов.

### Устойчивость по плотностям

Проверка устойчивости проводилась на плотностях `ratio ∈ {4.1, 4.2, 4.35, 4.5}` с размерами `n ∈ {500, 1000}` и seeds `{42, 123}` (по 2 инстанса на seed).

Тестировались два режима работы алгоритма: `baseline` с флагом `--disable-primal-dual` и `candidate` с включенным primal-dual (default конфигурация).

## Ядро make/break scoring

```cpp
// Вычисление вероятности флипа для кандидата v
const double adjusted_break = eps_ + break_count + penalty;
const double make_factor = 1.0 + make_weight_ * make_count;
const double probability = make_factor * std::pow(adjusted_break, -cb_);
```

Константа `eps_` представляет собой малую величину (≈ 1e-6) для численной стабильности. Параметр `make_weight_` определяет вес награды за make, в то время как `cb_` задает экспоненту штрафа за break. Дополнительный штраф за novelty добавляется через `penalty`.

### Дуальное обновление веса клаузы

```cpp
// Адаптивный шаг обучения (уменьшается со временем)
double eta_t = dual_eta_ / std::sqrt(static_cast<double>(step) + 1.0);

// Градиент: 1 если клауза осталась UNSAT, 0 если стала SAT
double grad = unsat_mask[chosen_clause] ? 1.0 : 0.0;

// Регуляризованный градиент (штраф за отклонение от веса 1.0)
double regularized_grad = grad - dual_reg_ * (old_weight - 1.0);

// Обновление с clipping в допустимый диапазон
double new_weight = std::clamp(
    old_weight + eta_t * regularized_grad,
    dual_weight_floor_,  // обычно 0.1
    dual_weight_cap_     // обычно 10.0
);
```

Параметр `dual_eta_` задает начальный шаг обучения (типично в диапазоне 0.1-0.5), `dual_reg_` определяет коэффициент регуляризации (обычно 0.01-0.1), а границы весов устанавливаются через `dual_weight_floor_` и `dual_weight_cap_`.

## Открытые вопросы и roadmap до финальной версии

### Что еще не закрыто

Сейчас доказано преимущество на dense-режимах. Требуется проверка отсутствия регрессий на sparse-задачах для полной уверенности в универсальности подхода.

Текущая выборка составляет 2-4 инстанса на конфигурацию. Для полноценной публикации необходимо расширить до ≥10 seeds с вычислением доверительных интервалов, что позволит делать более надежные статистические выводы.

В настоящее время оптимальные параметры зависят от размера задачи. Цель состоит в разработке универсальной конфигурации, которая будет работать эффективно без ручной настройки под каждый размер.

### План до финального релиза

Планируется добавить seeds 200-210 для размеров n=2000, 2500, 3000 с последующим вычислением 95% доверительных интервалов для более строгого статистического анализа.

Необходимо формализовать связь primal-dual динамики с subgradient descent и доказать либо опровергнуть наличие bounded regret.

Запланирована реализация adaptive bucket selection на основе параметров n и m/n с последующим A/B тестом против ручного тюнинга для проверки эффективности автоматического подхода.

### Разделение на "доказано" и "гипотезы"

Экспериментально подтверждено, что prototype algorithm 8C2 превосходит probSAT и W-WalkSAT на ratio=4.267, n=1500-3000. Установлено, что primal-dual механизм дает прирост на dense-режимах без регрессий. Показана критическая важность zero-break prioritization для достижения конкурентных результатов.

Предполагается, что преимущество сохраняется при n>3000, но это требует дополнительной экспериментальной проверки. Рассматривается возможность создания универсальной конфигурации без size-specific тюнинга. Исследуются теоретические гарантии сходимости (bounded regret).

## Приложение: Глоссарий терминов

| Термин              | Определение                                                                                   |
| ------------------- | --------------------------------------------------------------------------------------------- |
| **SAT**             | Boolean Satisfiability Problem — задача нахождения присваивания, удовлетворяющего CNF-формулу |
| **CNF**             | Conjunctive Normal Form — формула в виде конъюнкции дизъюнкций                                |
| **3-SAT**           | Частный случай SAT, где каждая клауза содержит ровно 3 литерала                               |

| **make/break**      | make = клаузы, ставшие SAT; break = клаузы, ставшие UNSAT после флипа                         |
| **Primal-Dual**     | Методы оптимизации, работающие одновременно с прямой и двойственной задачей                   |
